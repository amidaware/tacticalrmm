# Local Development Setup

Changes required to run TacticalRMM API locally (outside Docker/K8s).

Reference commit: `dd9bdbde` ("For local")

---

## 1. `settings.py` — Local defaults

**File:** `api/tacticalrmm/tacticalrmm/settings.py`

Set `DEBUG = True` and add `ADMIN_URL`:

```python
DEBUG = True        # was: False
ADMIN_URL = "admin/"  # add this line
```

Add local fallback defaults inside the `if not DOCKER_BUILD:` block, **before** the `TRMM_ROOT_DOMAIN` line:

```python
if not DOCKER_BUILD:
    # Defaults for local run without local_settings.py
    if not ALLOWED_HOSTS:
        ALLOWED_HOSTS = ["localhost", "127.0.0.1"]
    if not CORS_ORIGIN_WHITELIST:
        CORS_ORIGIN_WHITELIST = ["http://localhost:8080", "http://127.0.0.1:8080"]
    if "SECRET_KEY" not in dir():
        SECRET_KEY = "dev-secret-key-not-for-production"
    if "DATABASES" not in dir() or DATABASES.get("default", {}).get("ENGINE") == "django.db.backends.dummy":
        DATABASES = {
            "default": {
                "ENGINE": "django.db.backends.postgresql",
                "NAME": "tacticalrmm",
                "USER": "postgres",
                "PASSWORD": "postgrespass",
                "HOST": "127.0.0.1",
                "PORT": "5433",
            }
        }
    OPENFRAME_MODE = True

    TRMM_ROOT_DOMAIN = ...  # existing code continues
```

### Why

Without `local_settings.py` (which is generated by the installer), Django has no DB config, no SECRET_KEY, and no ALLOWED_HOSTS. These defaults let you run `manage.py runserver` immediately with just a local PostgreSQL.

## 2. `celery.py` — Don't block on missing Redis

**File:** `api/tacticalrmm/tacticalrmm/celery.py`

Add after `broker_connection_retry_on_startup = True`:

```python
# Don't block startup if Redis is unavailable (local dev)
app.conf.broker_connection_timeout = 2
app.conf.broker_connection_max_retries = 0
```

### Why

By default Celery retries Redis connection indefinitely, blocking `manage.py runserver` if Redis is not running. These settings make it fail fast.

## 3. `core/apps.py` — Disable Redis agent listener

**File:** `api/tacticalrmm/core/apps.py`

Comment out the agent update listener in `CoreConfig.ready()`:

```python
def ready(self):
    # Redis agent listener disabled for local development without Redis
    # if not settings.DEBUG or os.environ.get('RUN_MAIN') == 'true':
    #     try:
    #         from core.agent_updater import start_listener
    #         start_listener(daemon=True)
    #     except Exception as e:
    #         import logging
    #         logger = logging.getLogger(__name__)
    #         logger.error(f"Failed to start agent update listener: {str(e)}")
    pass
```

### Why

`start_listener()` opens a Redis pub/sub connection on app startup. Without Redis running locally, this throws an exception and prevents Django from starting.

## 4. `agents/utils.py` — Python 3.9 compatibility

**File:** `api/tacticalrmm/agents/utils.py`

Replace `match/case` (Python 3.10+) with `if/elif`:

```python
# Was (Python 3.10+):
match arch:
    case "amd64":
        arch_id = MeshAgentIdent.LINUX64
    ...

# Now (Python 3.9 compatible):
if arch == "amd64":
    arch_id = MeshAgentIdent.LINUX64
elif arch == "386":
    arch_id = MeshAgentIdent.LINUX32
elif arch == "arm64":
    arch_id = MeshAgentIdent.LINUX_ARM_64
elif arch == "arm":
    arch_id = MeshAgentIdent.LINUX_ARM_HF
else:
    arch_id = "not_found"
```

### Why

If your local Python is 3.9, `match/case` causes a `SyntaxError`. This change makes it backward compatible.

## Prerequisites

| Dependency | Required | Notes |
|---|---|---|
| PostgreSQL | Yes | Running on `127.0.0.1:5433`, DB `tacticalrmm`, user `postgres`, password `postgrespass` |
| Redis | No | Optional — Celery tasks and agent listener won't work, but API endpoints work fine |
| NATS | No | Optional — agent communication won't work, but API CRUD works |

## Quick start

```bash
cd api/tacticalrmm

# Apply migrations
python manage.py migrate

# Create superuser
python manage.py createsuperuser

# Run dev server
python manage.py runserver
```

API available at `http://localhost:8000/`
Admin panel at `http://localhost:8000/admin/`
